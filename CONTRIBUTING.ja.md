# i18next-turbo へのコントリビュート

i18next-turbo への貢献に興味をお持ちいただきありがとうございます。このドキュメントでは、コントリビュートのガイドラインと手順を説明します。

## 行動規範

本プロジェクトは行動規範を採用しています。参加するすべてのコントリビューターがこれを守ることに同意したものとみなします。参加前に [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md) を必ずお読みください。

## コントリビュートの方法

### バグの報告

バグを発見した場合は、以下の内容を含む Issue を作成してください。

- 明確で説明的なタイトル
- 問題の再現手順
- 期待される動作
- 実際の動作
- お使いの環境（OS、Rust バージョンなど）
- 関連するエラーメッセージやログ

### 機能の提案

機能の提案を歓迎します。以下の内容を含む Issue を作成してください。

- 機能の明確な説明
- ユースケースと例
- その機能が有用である理由

### プルリクエスト

1. **リポジトリをフォーク**し、`main` からフィーチャーブランチを作成する
2. **変更を行う**際は、下記のコーディング規約に従う
3. 新機能やバグ修正には**テストを追加**する
4. 必要に応じて**ドキュメントを更新**する
5. **テストを実行**し、すべて通ることを確認する
6. **プルリクエストを提出**し、明確な説明を記載する

## 開発環境のセットアップ

### 必要環境

- Rust 1.70 以降
- Cargo（Rust に同梱）

### ビルド

```bash
# リポジトリをクローン
git clone https://github.com/your-username/i18next-turbo.git
cd i18next-turbo

# プロジェクトをビルド
cargo build

# テストを実行
cargo test

# 例として実行
cargo run -- extract
```

## コーディング規約

### Rust のスタイル

- [Rust API Guidelines](https://rust-lang.github.io/api-guidelines/) に従う
- `cargo fmt` でコードをフォーマットする
- `cargo clippy` でよくある問題をチェックする
- 意味のあるコミットメッセージを書く

### コードの構成

- 関数は単一の責務に集中し、小さく保つ
- 複雑なロジックにはコメントを追加する
- 変数名と関数名は説明的にする
- 既存のコードパターンに従う

### テスト

- 新機能にはユニットテストを書く
- エッジケースとエラー条件をテストする
- PR を提出する前にテストが通ることを確認する

## コミットメッセージ

[Conventional Commits](https://www.conventionalcommits.org/) の形式に従ってください。

```
<type>(<scope>): <subject>

<body>

<footer>
```

タイプ:
- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメントの変更
- `style`: コードスタイルの変更（フォーマットなど）
- `refactor`: リファクタリング
- `test`: テストの追加または更新
- `chore`: メンテナンス作業

例:
```
feat(extractor): add support for Trans component children

Extract translation keys from Trans component children when
i18nKey is not specified.

Closes #123
```

## レビュープロセス

1. すべての PR には少なくとも 1 人のメンテナのレビューが必要です
2. CI が通ること（テスト、リント、フォーマット）
3. メンテナから変更を求められる場合があります
4. 承認後、メンテナがマージします

## 質問がある場合

質問がある場合は Issue を立てるか、メンテナまでお気軽にご連絡ください。
